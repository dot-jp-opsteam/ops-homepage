# 🐙 はじめての GitHub（初心者向け 使い方ガイド）

「GitHubってよく分からない…」という人向けに、**この作業に必要な分だけ**やさしくまとめました。
これだけ読めば、編集して公開サイトに反映するまで一人でできます。

---

## GitHubって何？（30秒のたとえ）

> みんなで1つの作品を作る **共有の作業部屋** だと思ってください。
> - **GitHub（ネット上）** ＝ みんなの完成版が置いてある「本棚」
> - **あなたのPC（ローカル）** ＝ 自分専用の「作業机」
>
> 流れは「本棚から最新を**持ってくる** → 机で**編集** → 本棚に**戻す**」だけ。

---

## 使う道具（3つ）

1. **GitHub Desktop** … 本棚⇔机を行き来する **運び屋**（Pull / Commit / Push をする）
2. **VS Code** … HTMLを **編集** するエディタ
3. **Live Preview**（VS Codeの拡張機能）… 編集中の見た目を **その場で確認**

---

## 最初の1回だけ：クローン（机に持ってくる）

「クローン」＝ GitHubの本棚から、自分のPCに**まるごとコピーを持ってくる**こと。

1. GitHub Desktop で **File → Clone repository**
2. `dot-jp-opsteam/ops-homepage` を選ぶ
3. 保存先を選んで Clone
   - ⚠️ **OneDriveの外**（例：`C:\Users\自分\github\`）に置くのがおすすめ（同期トラブル防止）

> これは最初の1回だけ。2回目以降は下の「毎回のサイクル」を回すだけです。

---

## 毎回の基本サイクル（この5歩を覚えればOK）

```
① Pull → ② 編集 → ③ 保存 → ④ Commit → ⑤ Push
```

| 歩 | 何をする | たとえ |
|---|---|---|
| **① Pull** | GitHub Desktop で「Pull origin」を押す | 本棚から**最新を持ってくる**（古い状態で始めない） |
| **② 編集** | VS CodeでHTMLを直す（Live Previewで確認） | 机で**作業する** |
| **③ 保存** | `Ctrl + S` | こまめにセーブ |
| **④ Commit** | 左下に要約を書いて「Commit to main」 | 変更に**名前を付けて記録**（チェックポイント） |
| **⑤ Push** | 上の「Push origin」を押す | 本棚に**戻す＝公開サイトに反映**（数十秒で反映） |

> 💡 **作業を始める前に Pull、終わったら Push**。これさえ守れば事故はほぼ起きません。

---

## ⚠️ もし他の人と編集がぶつかったら（コンフリクト）

「自分が編集している間に、誰かが先にPushしていた」場合の話です。**データが消えることはありません**。

### あなたがPushしようとすると…
GitHubが「先に最新を取り込んでね」と**止めてくれます**（勝手に上書きしない安全装置）。
→ 言われたとおり **「Pull origin」** を押します。すると2パターン：

**パターンA：ぶつからない（ほとんどコレ）✅**
あなたと相手が**違うファイル／違う場所**を編集していた
→ Gitが**自動で合体**。そのまま **Push** で完了。

**パターンB：ぶつかる（コンフリクト）⚠️**
**同じファイルの同じ行**を二人が別々に書き換えていた時だけ
→ GitHub Desktopが「衝突したファイルがあるよ」と表示
→ VS Codeで開くと両方の変更が並ぶので、**残す方をクリックで選ぶ** → 保存 → Commit → Push

> 🛡 **「担当ページ中心に触る」ルールを守れば、パターンBはほぼ起きません**（みんな別ファイルだから自動合体になる）。
> もし衝突画面が出たら、慌てず**スクショしてまとめ役に相談**でOK。

## 用語ミニ辞典

| 言葉 | 意味（ざっくり） |
|---|---|
| **リポジトリ** | プロジェクト一式（このサイトのフォルダ全体） |
| **クローン** | 本棚→自分のPCにコピーを持ってくる（最初の1回） |
| **Pull** | 最新を取り込む（持ってくる） |
| **Commit** | 変更に名前を付けて記録（チェックポイント） |
| **Push** | 自分の変更を本棚に戻す＝公開反映 |
| **ブランチ** | 本番を触らずに作業する「別レーン」。大きな変更の時に使う |
| **マージ** | 別レーンの変更を本番（main）に合流させる |
| **コンフリクト** | 同じ場所を二人が変えてGitが迷う状態（人が選んで解決） |

---

## 困ったときの3か条

1. **慌ててボタンを連打しない**（特に赤字の警告が出たら一度手を止める）
2. **分からない画面はスクショ**して、まとめ役に聞く
3. **Pushする前にPull** を思い出す

> Gitは「やり直し」がきく仕組みです。多少間違えても直せるので、こわがらずに触ってOK 👍
