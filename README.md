## 🛠 みんなが使う道具

1. **GitHub Desktop** ＝ パソコンとネット(GitHub)をつなぐ"運び屋"
2. **VS Code** ＝ HTML/CSSを編集するエディタ（＋拡張機能「Live Preview」で見た目を確認）

## 🔄 編集の基本の流れ（これだけ覚えればOK）

**最初の1回だけ：** GitHubから自分のPCに「クローン」（コピーを持ってくる）

**毎回の作業：**

```
① Pull（最新を取り込む）→ ② VS Codeで編集 → ③ 保存
   → ④ Commit（変更に名前をつける）→ ⑤ Push（GitHubへ送る）
```

👉 **Push すると、数十秒で公開サイトに自動反映**されます。

## 👥 担当（自分のページ中心に触る）

- **こうへい**：存在意義（mission）・メンバー紹介（members）
- **レアンドロ**：今後の方針（future）・意見箱（contact）
- **そうま**：活動（activities）・成果物（works）

---

## 🎨 ③ デザイン統一でやること（次のステップ）

- **レアンドロのデザインを基準**に、全ページの見た目をそろえます
- まず**共通の「枠」**（ヘッダー・色・フォント・ボタン）を全ページにそろえる → これだけで「同じサイト」感が出る
- ※ボタンの動きなど**細かいアニメは今回は入れません**（パッと見の統一感が目標）

## ✨ ④ 各ページの仕上げ

- 共通の枠が整ったら、**各自が自分のページを自由に作り込めます**
- 文章・写真・レイアウトを理想の形に。自分専用のデザインも足せます
- **枠（ヘッダー・色）は共通のまま** → 統一感を保ちつつ個性も出せる

---

## 📌 チームのお約束（事故防止）

1. **作業の前に必ず Pull、終わったら Push**
2. **自分のページは自由に編集OK**
3. **みんな共通のファイル（共通CSS・ヘッダー・index など）を触るときは、一声かけてから**
4. デザインは「**枠は共通／中身は各自の個性OK**」
