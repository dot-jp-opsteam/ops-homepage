# ✏️ 編集ガイド（メンバー向け）

このサイトは **1つの共通デザイン（leandroデザイン）** で全ページが揃っています。
ここを守れば、誰が触っても見た目が崩れません。AIにコーディングを頼む人は
このページ末尾の **「AIに渡すプロンプト」** をコピペしてから依頼してください。

---

## 30秒でわかる仕組み

- 見た目（色・背景・レイアウト）は全部 **`css/style-leandro.css`** が決めている
- 動き（ふわっと出現・スマホメニュー・流れる文字）は **`js/main-leandro.js`** が動かしている
- 各HTMLファイルは、この共通の **部品（クラス名）** を並べているだけ
- だから **CSS / JS / 共通部品の「形」を変えると、全ページ同時に壊れる**

あなたが変えるのは基本 **「タグの中の文章」だけ** です。

---

## 🟢 変えてOK（自由に編集していい）

- タグに **はさまれた文章**（見出し・説明・名前・役割・Q&Aの答え など）
- カードを **増やす／減らす**
  → `<div class="wash"> 〜 </div>` の **かたまりごとコピペ** して、中の文章だけ書き換える
- メンバーページの **写真**
  → 画像を `images/` フォルダに入れて、`<img src="...">` のファイル名を差し替え
- 自分の担当ページの `lead`（リード文）や `ttl`（見出し）の文章

---

## 🔴 変えてはいけない（触ると全ページ崩れる）

| # | 触らないもの | なぜ |
|---|---|---|
| 1 | **`css/style-leandro.css`** と **`js/main-leandro.js`** の中身・ファイル名 | 全13ページ共通。1文字変えると全部に影響 |
| 2 | **`<head>` の `<link rel="stylesheet" ...>` と フォントの行** | デザインの読み込み元。消すと真っ白になる |
| 3 | ページ上部の **`<svg class="blob-defs">…</svg>` のかたまり** | 背景のグニャ図形の“型紙”。消すと背景が出ない |
| 4 | **ヘッダー `<header class="header">` とフッター `<footer class="footer">`** | 全ページ同じにする約束。**唯一OKなのは、今いるページのメニューに `class="current"` を付け替えることだけ** |
| 5 | **クラス名**（`section` `container` `ttl` `wash` `page-kv` `reveal` `cta-band` など） | 見た目・動きの“あいことば”。リネーム＝崩壊 |
| 6 | ページ最後の **`<script src="...main-leandro.js"></script>`** | これが動きを起動している |
| 7 | **`contact.html` のフォーム**：`id="fb-form"` と各入力欄の `name="name / cat / msg / contact"` | Googleフォーム送信と直結。変えると意見が届かなくなる |
| 8 | **HTMLファイルの名前・置き場所**（`members/` から出さない 等） | ページ間のリンクが全部切れる |

> 補足：`reveal` はスクロールで“ふわっ”と出すための合図です。新しいかたまりを足すときは
> `class="... reveal"` を付けると同じ動きになります。

---

## ⚠️ パスの注意（ここだけ間違えやすい）

ファイルの置き場所で、リンクや画像の“道順”が変わります。

| どこを編集中？ | CSS / JS / 画像の書き方 | 他ページへのリンク |
|---|---|---|
| **トップ階層**（`index.html`, `mission.html`, `activities.html` など） | `css/...` `js/...` `images/...` | `members.html` のように **そのまま** |
| **`members/` の中**（`member01.html` 〜） | `../css/...` `../js/...` `../images/...` | `../mission.html` のように **頭に `../`** |

`../` は「1つ上のフォルダに戻ってから探す」という道順です。

> GitHub Pages は **大文字・小文字を区別** します。画像は `OPS-logo.png` のように
> 実際のファイル名と完全に一致させてください（`ops-logo.png` だと表示されません）。

## 🤖 AIにコーディングを頼むときのプロンプト（コピペ用）

> 編集したいHTMLファイルを開いた状態で、下の文をそのまま貼ってから「○○を△△に直して」と頼んでください。

```text
あなたはこのプロジェクトの編集を手伝うアシスタントです。
このサイトは「leandroデザイン」という1つの共通デザインで全ページが統一されています。
編集するときは、次のルールを必ず守ってください。

【絶対に変えないこと】
1. css/style-leandro.css と js/main-leandro.js は編集も改名もしない（全ページ共通のため）。
2. <head> 内の stylesheet とフォントの <link>、ページ上部の <svg class="blob-defs"> のブロック、
   ページ末尾の <script src="...main-leandro.js"> は消さない・動かさない。
3. <header class="header"> と <footer class="footer"> は全ページ共通なので構造を変えない。
   許されるのは「今いるページのメニュー項目に class="current" を付け替える」ことだけ。
   メニューの項目や並び順（Mission→Members→Activities→Works→Future→Contact）は変えない。
4. 既存のクラス名（section, container, ttl, wash, page-kv, reveal, cta-band, footer など）は
   リネームしない。新しい独自CSSやstyle属性での大きな見た目変更もしない。
5. contact.html のフォームは id="fb-form" と各入力欄の name 属性
   （name / cat / msg / contact）を変えない（Googleフォーム送信に直結しているため）。
6. HTMLファイルの名前や置き場所は変えない。

【やっていいこと】
- タグにはさまれた文章（見出し・本文・名前・役割・Q&Aの答えなど）の編集。
- カードを増やすときは <div class="wash"> 〜 </div> のブロックを丸ごとコピーして中の文章だけ変える。
  新しいブロックには既存と同じく class に reveal を付ける。
- 画像は images/ フォルダに置き、src のファイル名を差し替える。

【パスのルール】
- トップ階層のページ（index.html, mission.html など）: css/... js/... images/... と書く。
- members/ フォルダ内のページ: ../css/... ../js/... ../images/... と書き、
  他ページへのリンクも ../mission.html のように頭に ../ を付ける。
- 画像名は大文字小文字も含めて実ファイルと完全一致させる（GitHub Pagesは区別する）。

以上を守ったうえで、既存ページと同じ見た目・トーンを保って編集してください。
変更後は「どの部分をどう変えたか」を一言で説明してください。
```

## 🔄 編集の流れ（毎回これ）

1. **Pull**（始める前に最新を取り込む）
2. VS Code で自分の担当ページを編集 → 保存 → **Live Preview** で確認
3. **Commit**（変更にメモを付けて記録）
4. **Push**（GitHubへ送る → 公開サイトに自動反映）

### チームのお約束
- 触るのは **自分の担当ページ中心**
- **共通ファイル**（`index.html`・ヘッダー/フッター・`css/`・`js/`）を変えたいときは、
  先にチームに一声かける（全員に影響するため）
- 始める前に Pull、終わったら Push（ぶつかり防止）
